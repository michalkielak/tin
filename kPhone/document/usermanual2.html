KPhoneIS User Manual: The Commandline

To get help, write kpsi -h

kfl@papa:~> kpsi -h


kpsi [-u <user>] [-a <host>] [-p <port>[:n]] [-t] [-r] [-n] 

Options


-u <user> user identification in database
 <user> is not the user part of the SIP-URI (SIP-Phone number), but the user's ID in the database ($HOME/.qt/kphonesirc)

-a <host> dissipate-host
 <host> the IP address of the interface used by the phone (i.e. your computer's address). If this parameter is not set, KPhoneSI scans the interface table. Th user is ask to make the choice.

-p <port>[:n] port number you listen on
 <port> The listen-to port. It is used to address the UA from outside. Your value is stored in the settings. The value 0 resets the port to 5060, what is also the default, if the setting is not set. If your value is in use KPhoneSI finds a new one greater then your value. If the port of your choice is used, n gives the allowed numbers of retrys to find a new port. The value 0 allows only your port.

-l loose route
 
-s strict route
 
-r send requests to the To-URI directly
 
-t <nr>  UA point to point
 IPv4: 4 and IPv6: 6 are legal values In this case the dissipate-host equals the host part of sip URI of the UA). -t is one possible method to start point to point sessions without involving a proxy. The registration is switched off and the SIP-URI takes the form sip:port@dissipate-host:port.

-n do NOT register after startup
 Usefull if you want to configure the phone and to avoid startup with unknown parameters
 
-v n  n=1 SIP messages  n=2 debug info n=3 both
